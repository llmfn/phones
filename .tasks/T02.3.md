# JS modules — wire the request/response round-trip

## DESCRIPTION

Implement the browser-native ES modules under `static/js/` so a query does a real
round-trip and renders results. Module split per `docs/specs.md`:

- `state.js` — in-memory state object + localStorage helpers for `llmfn_current_layer`,
  `llmfn_phase_state`, `llmfn_chat_history`; generate/persist a `session_id`;
  derive `active_layers = [1..current_layer]`.
- `api.js` — `recommend(payload)` POSTs JSON to `/api/recommend` (same-origin),
  returns parsed JSON, surfaces network/HTTP errors for the error bubble.
- `render.js` — render chat history (bubbles), product cards (empty for now),
  inline error bubble, and the trace timeline (badge + name + latency, expandable
  raw input→output JSON, color/skip classes).
- `events.js` — bind Send, the layer dropdown (update current_layer → re-derive
  active_layers → persist → re-render trace state), Reset to defaults, Simulate API
  Failure (one-shot flag for the next request), Copy trace as JSON.
- `app.js` — on load: hydrate state from localStorage, render initial chat +
  controls, attach events.

## ACCEPTANCE CRITERIA

- [x] Sending a query appends a user bubble, calls the API, and renders the
      assistant answer bubble
- [x] The trace timeline renders one row per layer with status color/skip styling
      and expandable input→output JSON
- [x] Layer dropdown updates `current_layer`/`active_layers` and persists
- [x] Simulate API Failure applies only to the next request; failures show an
      inline error bubble
- [x] Copy trace as JSON copies the latest trace to the clipboard
- [x] Chat history and selected layer survive a page reload
