# Flask app + uv project + stub /api/recommend

## DESCRIPTION

Create the `uv`-managed Flask app (`app.py`) that serves the UI and the stub API,
same-origin.

- `GET /` → `render_template("index.html")`.
- `POST /api/recommend` → read the request body (`query`, `current_layer`,
  `active_layers`, `phase`, `simulate_failure`, `session_id`) and return the
  response shape from the contract: `{answer, products, trace}`.
- A `LAYERS` list `(n, name)` is the single source of layer names:
  Search, Prompt, Schema, Context, State, Memory, Tool Use, Evals.
- Stub trace: one row per layer 1–8; rows whose layer is in `active_layers` →
  `status: "success"` with placeholder `input`/`output` and a small fake
  `latency_ms`; later layers → `status: "skipped"`. `answer` is a canned echo.
- `simulate_failure: true` → error-shaped response (a row with `status: "error"`
  and an `answer` describing the simulated failure).

## ACCEPTANCE CRITERIA

- [x] `app.py` exists; `uv run flask --app app run` boots on port 5000
- [x] `GET /` returns the rendered `index.html`
- [x] `POST /api/recommend` returns `{answer, products: [], trace: [...]}` matching
      `docs/specs.md` field names
- [x] Trace has 8 rows: layers in `active_layers` are `success`, later layers
      are `skipped`
- [x] `simulate_failure: true` returns an error-shaped response with an `error`
      trace row
