# Set up task tracking for Claude

## DESCRIPTION

Establish a lightweight, file-based task system in `.tasks/` that Claude can
read, create, and update, and document it in `CLAUDE.md` so the workflow is
consistent across sessions.

Each task is a single markdown file with `status` + `depends_on` frontmatter, a
DESCRIPTION, and ACCEPTANCE CRITERIA. Tasks may have subtasks (`T01.1`, `T01.2`).

## ACCEPTANCE CRITERIA

- [x] `.tasks/` exists in the repo root with at least one sample task
- [x] `CLAUDE.md` exists in the repo root and explains the task system
- [x] Claude can list, read, create, and update tasks following the documented format
