# Build the minimal app skeleton

## DESCRIPTION

Stand up the single Python (Flask) web app described in `docs/specs.md`: one app
that serves both the two-panel UI (plain HTML/CSS/browser-native JS modules) and
the `POST /api/recommend` backend, same-origin.

This is PLAN.md Task 1 — a *skeleton*. The endpoint is a **stub**: it returns the
response shape from the API contract with a fabricated trace (active layers
`success`, later layers `skipped`) and a canned echo answer. No real
recommendation logic, dataset, or product data yet — that is `T03` (Layer 1).

The skeleton must prove the full round-trip: typing a query sends a POST, and the
UI renders the answer bubble plus the trace timeline, with the layer dropdown,
reset, simulate-failure, and copy-trace controls all wired.

Dependencies are managed with `uv` (`pyproject.toml` + `uv.lock`); run via
`uv run`.

## ACCEPTANCE CRITERIA

- [x] `uv run flask --app app run` starts the app on `http://127.0.0.1:5000`
- [x] `GET /` serves the two-panel UI shell (App Canvas | Control + Trace)
- [x] `POST /api/recommend` returns `{answer, products, trace}` per the contract,
      with an 8-row trace (active layers `success`, later layers `skipped`)
- [x] `simulate_failure: true` produces an error-shaped response
- [x] Sending a query in the browser renders user + assistant bubbles and the
      trace timeline; layer dropdown, reset, simulate-failure, and copy-trace work
- [x] Selected layer and chat history persist across reload (localStorage)
